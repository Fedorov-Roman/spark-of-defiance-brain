# 👥 Team Charter

## Основная команда (Active Pipeline)

### Kimi — Architect / Tech Lead
- **Роль:** Единственный проектировщик. Архитектурные схемы, ТЗ на каждый CP, код-ревью, интеграция, коммуникация с Романом.
- **Что делает:** Пишет ТЗ, каркасы классов, определяет Scene Tree, AutoLoad, сигналы, проверяет Godot API.
- **Что НЕ делает:** Не пишет production-ready `.gd` / `.tscn`. Не принимает решений без контекста.
- **Критерий успеха:** Builder получает ТЗ, из которого невозможно сделать не так.

### DeepSeek v4 Pro — Primary Builder
- **Роль:** Единственный, кто пишет production-код игры.
- **Что делает:** Реализует полные файлы `.gd`, `.tscn`, `.tres` по ТЗ Kimi. Self-review перед сдачей.
- **Что НЕ делает:** Не проектирует архитектуру, не пишет ТЗ, не принимает решений о нодах/классах.
- **Критерий успеха:** Код проходит ревью Kimi с минимумом KP (критических проблем).

### Роман — Заказчик / Product Owner / Leonardo Operator
- **Роль:** Владелец продукта, тестировщик, оператор Leonardo.Ai, геймдизайнер (концепт).
- **Что делает:** Даёт ответы на вопросы Architect'а, тестирует CP по чеклистам, генерирует арты в Leonardo по промптам Kimi, управляет резервом.
- **Что НЕ делает:** Не пишет код (если не является Builder'ом), не принимает архитектурных решений в обход Kimi.

### Leonardo.Ai Free Plan — Lead Concept Artist & Texture Designer
- **Роль:** Генерация концептов, фонов, текстур, UI-иконок.
- **Ограничения:** 150 Fast токенов/день, только Fast-режим, размер до 1024×1024, публичные генерации.
- **Модели:** Phoenix 1.0 (персонажи), Cinematic Kino (фоны), Concept Art (текстуры), Graphic Design (UI), FLUX Schnell (черновики).
- **Правило:** Промпты визуально самодостаточные, без имён персонажей/фракций/игровых терминов.
- **Что НЕ делает:** Не создаёт спрайтовую анимацию, true pixel-art, шрифты. Только концепты и референсы.

---

## Резерв (Escalation Pool)

### DeepSeek v4 Flash — Researcher / Backup Builder
- **Роль:** Поиск Godot API, консультации, вспомогательные тулзы, замена Primary Builder при блоке.
- **Когда задействован:** По запросу Kimi. Не самостоятельно.
- **Что делает:** Ищет методы в документации, предлагает 3 варианта решения, пишет тулзы (не runtime-код).
- **Что НЕ делает:** Не пишет код в основной проект без ТЗ Kimi.

### Qwen 3.7 Max — Reserve Architect
- **Роль:** Сложная архитектурная экспертиза, если Kimi не может выбрать между паттернами.
- **Когда задействован:** По запросу Kimi. Не самостоятельно.
- **Что делает:** Анализирует архитектурные альтернативы, даёт обоснованные рекомендации.
- **Что НЕ делает:** Не пишет ТЗ, не общается с Романом, не управляет Builder'ами.

---

## Процесс коммуникации

```
Роман ←→ Kimi (Architect)
            ↓
    DeepSeek Pro (Builder)
            ↓
    Код → Ревью Kimi ←→ Роман (тест)
            ↓
    [При блоке] → Flash / Qwen (Reserve)
```

**Правило:** Reserve не общается с Романом напрямую. Вся коммуникация через Kimi.

---

## Протокол эскалации

1. **Primary Builder застрял (3 итерации ревью)** → Kimi переключает задачу на Flash.
2. **Architect не может выбрать паттерн** → Kimи запрашивает Qwen.
3. **Нестандартная механика** → Kimi запрашивает обоих резервистов на анализ.
4. **Оптимизация FPS** → Kimi запрашивает Flash на профайлинг.
