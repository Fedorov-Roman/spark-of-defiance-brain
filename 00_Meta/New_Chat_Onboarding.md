## 3. Контекст проекта (сжатый)

**Игра:** 2D стелс-платформер с метроидванией. Пиксель-арт 32×32, масштаб x2/x3.  
**Герой:** Кай («Искра»), выживший на планете Постулат. Стелс, акробатика, «Кинжал Времени» (замедление мира до 30%, Кай на 100%).  
**Прогрессия:** Спасение 6 NPC-мастеров в хабе открывает способности (двойной рывок, крюк, замедление, карта).  
**Структура:** 3 зоны (Красные Дюны, Полярный Оазис, Глубинные Лаборатории) + 3 босса + хаб.  
**Сохранение:** JSON, полное состояние мира, автосейвы на чекпоинтах.  
**Управление:** Геймпад + клавиатура/мышь.  
**Язык:** Английский.

## 4. Ключевые архитектурные решения Godot

- **Корень игрока:** `CharacterBody2D` с `AnimatedSprite2D` (позже `AnimationTree`).
- **Уровни:** `TileMap` (grid) + отдельные сцены для интерактива/врагов.
- **Физика:** `_physics_process` для движения, `_process` для UI/визуала.
- **Стелс:** `Area2D` (конус зрения) + `RayCast2D` (проверка линии видимости) + группы `"enemies"`, `"traps"`.
- **Время:** Кастомный `time_scale` через модификацию `delta` для группы `"time_affected"`. `Engine.time_scale` НЕ используется (сломает UI).
- **Сохранение:** `SaveManager` (AutoLoad) → JSON в `user://save.json`.
- **Прогрессия:** `ProgressionManager` (AutoLoad) → `Dictionary` с флагами NPC.
- **Хаб:** Одна динамическая сцена, NPC появляются по флагам `visible`/`process`.
- **UI:** `CanvasLayer` с `Control` нодами, тема `Theme` для пиксель-арта.
- **Освещение:** `PointLight2D` + `CanvasModulate` (глобальная тьма) + `LightOccluder2D`.
- **Частицы:** `GPUParticles2D`.
- **Камера:** `Camera2D` с `smoothing`, `limit` по границам TileMap.

## 6. Правила для Builder'а (если ты Builder — читай)

1. Не принимай архитектурных решений. Если ТЗ неясно — задай уточняющий вопрос Architect'у.
2. Не пиши ТЗ. Только код по ТЗ.
3. Строгая типизация GDScript: `-> void`, `-> int`, `-> float`, `-> bool`, `-> Vector2`.
4. Используй `@export` для Inspector, `@onready` для кэширования нод.
5. `_physics_process` для физики, `_process` для UI/анимаций.
6. `Tween` создаётся через `create_tween()`, а не `new Tween()`.
7. `queue_free()` не удаляет мгновенно — не обращайся к ноде после него.
8. Сигналы: подключай в `_ready()`, отключай в `_exit_tree()`.
9. Не используй `get_node()` в `_process` — используй `@onready`.
10. Сдавай файлы `.gd`, `.tscn`, `.tres` полностью, не фрагментами.
