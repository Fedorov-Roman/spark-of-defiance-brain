# Team Charter

## 1. Миссия проекта

Создать **2D Stealth-Platformer Metroidvania** «Spark of Defiance» на Godot 4.6.3 для PC (Windows), с фокусом на:
- Хардкорный стелс с видимыми конусами зрения
- Акробатическое движение (wall-slide, dash, grapple)
- Метроидвания-прогрессию через спасение NPC
- Пиксель-арт эстетику 32×32 с динамическим освещением

Цель: портфолио для вуза + личный инди-релиз.

---

## 2. Роли и ответственность

### Роман (Заказчик / Геймдизайнер / Оператор)
- Формулирует видение игры
- Отвечает на архитектурные вопросы Kimi
- Генерирует арты в Leonardo.Ai по промптам Kimi
- Тестирует билды после каждого CP
- Принимает или отклоняет результаты ревью

### Kimi (Architect / Tech Lead)
- **Единственный проектировщик.** Пишет ТЗ на каждый CP.
- Исследует Godot API перед выдачей ТЗ.
- Проводит код-ревью Builder'а (KP/NC).
- Интегрирует модули в единую архитектуру.
- Пишет промпты для Leonardo.Ai.
- **Не пишет production-ready код.** Только каркасы, примеры (≤30 строк), схемы.

### DeepSeek v4 Pro (Primary Builder)
- **Единственный источник production-кода.**
- Реализует полные файлы `.gd`, `.tscn`, `.tres` по ТЗ Kimi.
- Проходит self-review перед сдачей.
- Исправляет критические проблемы (KP) из ревью Kimi.
- **Не проектирует архитектуру.** Не пишет ТЗ.

### DeepSeek v4 Flash (Reserve Builder / Researcher)
- **Не пишет код в основной проект без ТЗ Kimi.**
- Ищет Godot API, предлагает паттерны.
- Пишет вспомогательные тулзы (генераторы, парсеры).
- Заменяет Primary Builder при блоке (>3 итераций ревью).
- Работает только по прямому запросу Kimi.

### Qwen 3.7 Max (Reserve Architect)
- **Не принимает архитектурных решений без Kimi.**
- Консультирует при сложных архитектурных тупиках.
- Предлагает 3+ варианта решения с плюсами/минусами.
- Работает только по прямому запросу Kimi.

### Leonardo.Ai Free (Lead Concept Artist)
- Генерирует концепты, фоны, текстуры, UI-иконки по промптам Kimi.
- **Не знает контекста проекта.** Работает только по визуально самодостоятельным промптам.
- Оператор — Роман.

## 3. Процесс работы (CP Pipeline)

```
[CP-N]
   │
   ▼
Kimi: Исследует API → Пишет детальное ТЗ + каркасы
   │
   ▼
DeepSeek Pro: Реализует файлы по ТЗ
   │
   ▼
DeepSeek Pro: Self-review по чеклисту
   │
   ▼
DeepSeek Pro: Отправляет файлы Kimi
   │
   ▼
Kimi: Код-ревью (KP/NC)
   │
   ▼
DeepSeek Pro: Исправления (если нужны)
   │
   ▼
Kimi: Интеграция + проверка согласованности
   │
   ▼
Роман: Тестирует CP по чеклисту → сообщает баги
   │
   ▼
[CP-N+1]
```

## 4. Правила коммуникации

1. **Kimi — единственная точка контакта с Романом по архитектуре.**
2. **Builder не общается с Романом напрямую.** Все вопросы через Kimi.
3. **Роман не генерирует арты без промпта Kimi.** Leonardo работает только по ТЗ Architect'а.
4. **Резерв (Flash/Qwen) — только через Kimi.**
5. **При конфликте мнений:** аргументы в пользу простоты и надёжности. Если спор неразрешим — решает Роман.

## 5. Критерии качества

### Код (Builder)
- Строгая типизация GDScript (`-> void`, `-> int`, `-> Vector2`)
- `@export` для Inspector, `@onready` для кэширования нод
- `_physics_process()` для физики, `_process()` для визуала
- Сигналы (Signals) для decoupling, не прямые вызовы
- Нет `get_node()` в `_process` — только `@onready`
- Tween через `create_tween()`, не `new Tween()`
- `queue_free()` → нода удалится в конце кадра, не обращаться к ней сразу

### Арт (Leonardo)
- Промпты без имён персонажей/фракций (только визуальные описания)
- Generation Mode: Fast, Size: Medium/Small
- Negative Prompt всегда включён и заполнен
- Tiling: ON только для текстур

## 6. Эскалация

| Ситуация | Действие | Кто решает |
|----------|----------|------------|
| Builder застрял на 3+ итерациях | Переключить на Flash | Kimi |
| Архитектурный тупик (2+ паттерна) | Запросить Qwen | Kimi |
| Нестандартная механика | Flash + Qwen на анализ | Kimi |
| Оптимизация FPS | Flash ищет bottleneck | Kimi |
